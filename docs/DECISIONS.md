# Decisions

## D-001 — GitHub is the project source of truth

Date: 2026-06-24
Status: ACCEPTED

All tasks, implementation reports, reviews and current-state documentation are maintained in `xahinvest-DNA/Memorial_site`. Chat discussion is not a substitute for repository records.

## D-002 — Management model: Product Owner / Project Manager / Codex

Date: 2026-06-24
Status: ACCEPTED

- Андрей is Product Owner.
- ChatGPT is Project Manager.
- Codex is executor.

Codex receives tasks from the Project Manager, reports through Pull Requests, and does not independently expand product scope.

## D-003 — The product is purchase infrastructure, not a catalogue site

Date: 2026-06-24
Status: ACCEPTED

The product must support the transaction and fulfilment journey, not only promotion and lead capture.

## D-004 — MemorialCase is the conceptual system centre

Date: 2026-06-24
Status: PROVISIONALLY ACCEPTED

A single case should connect customer need, specification, designs, estimates, approvals, contract, production, installation and warranty. Final domain details require Discovery validation.

## D-005 — No application implementation before Product Gate

Date: 2026-06-24
Status: ACCEPTED

Repository documentation and Discovery work may proceed. Application scaffolding, UI implementation and final technical choices are blocked until the Product Owner passes the Product Gate.

## D-006 — Real media may arrive later; fabricated proof is prohibited

Date: 2026-06-24
Status: ACCEPTED

Design and content architecture may use clearly marked neutral placeholders. No stock or generated image may be presented as a completed company project, production facility, employee, material sample or customer evidence.

## D-007 — Human review remains at critical commitment points

Date: 2026-06-24
Status: PROVISIONALLY ACCEPTED

Exact technical suitability, final price, artistic proof, cemetery constraints and production commitment require human validation unless later evidence supports safe automation.
